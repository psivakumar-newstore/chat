This is an internal API for the customer support team, to re-index orders in the NewStore platform.
Reindex single customer order
Updates a
single
customer order if data is missing in NOM or an order needs a manual data
update, making the re-indexing of the existing order necessary.
Authorizations:
newStoreAccessToken
Request Body schema:
application/json
order_id
string
Unique order UUID of the order which should be indexed.
Responses
204
The indexing has been executed successfully.
400
Bad Request - Request data is invalid.
404
Order doesn't exist
409
Failed to execute index request due to a concurrenct request
500
An internal error occurred.
default
Unexpected Error
