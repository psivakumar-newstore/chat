This is an internal API for customer support to update the
service level
for orders in the NewStore platform.
Update service level
Sets a new
service level
for conflict items in the specified order.
Authorizations:
newStoreAccessToken
path
Parameters
order_id
required
string
The unique ID of the order. To retrieve the list of orders, use the
List orders
method.
Request Body schema:
application/json
new_service_level
required
string
non-empty
The chosen service level identifier.
Responses
200
The service level was updated successfully.
400
Bad Request Error.
409
The current order id does not contain any stuck orders.
500
Internal Server Error.
patch
/orders/{order_id}/service_level
/v0

Get routing options
Get shipping options for a given routing request containing:
delivery_destination
routing_strategy
products
flat_products
Note: The routing request is a nested dictionary object. The caller should construct the object,
serialize it to a query string and pass it in the URL query parameters with the routing_request key.
Authorizations:
newStoreAccessToken
query
Parameters
routing_request
object
A json encoded string with details for routing request
Responses
200
Successfully returning routing insights for the specified order
400
Client sent malformed request.
500
An internal error occurred.
